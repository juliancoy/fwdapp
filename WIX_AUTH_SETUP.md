# Wix Authentication Setup Guide

This guide explains how to set up Wix Members authentication with JWT tokens for the Civic Tech subdomain.

## Overview

**Architecture:**
- Main site: `marylandforwardparty.com` (Wix - handles login)
- Elections app: `elections.marylandforwardparty.com` (this HTML app)
- Authentication: JWT tokens generated by Wix Velo backend

**Flow:**
1. User logs in on main Wix site
2. Wix Members system authenticates them
3. They click "Go to Elections" → Velo generates a JWT
4. Redirect to elections app with `?token=<jwt>`
5. Elections app verifies token and recognizes the user

## Part 1: Wix Velo Backend (jwt.js)

Create a new backend file in your Wix site: **Backend > jwt.js**

```javascript
// Backend/jwt.js
import { Permissions, webMethod } from "wix-web-module";
import { authentication } from "wix-members-backend";
import jwt from "jsonwebtoken";

// Store this securely - use Wix Secrets Manager in production
const JWT_SECRET = "your-secret-key-here-change-this-in-production";
const TOKEN_EXPIRY = "1h"; // Token valid for 1 hour

/**
 * Generate a JWT token for the currently logged-in member
 * This is called from the frontend when they click "Go to Elections"
 */
export const generateMemberToken = webMethod(
  Permissions.Anyone, // Or Permissions.Member to require login
  async () => {
    try {
      // Get current member info
      const member = await authentication.getCurrentMember();
      
      if (!member._id) {
        return { success: false, error: "Not logged in" };
      }

      // Create JWT payload
      const payload = {
        memberId: member._id,
        email: member.loginEmail,
        name: member.contactDetails?.firstName 
          ? `${member.contactDetails.firstName} ${member.contactDetails.lastName || ""}`.trim()
          : member.loginEmail,
        // Add any roles/permissions from Wix
        roles: member.contactDetails?.labelKeys || [],
        // Standard JWT claims
        iat: Math.floor(Date.now() / 1000),
        exp: Math.floor(Date.now() / 1000) + (60 * 60), // 1 hour
        iss: "marylandforwardparty.com",
        aud: "elections.marylandforwardparty.com"
      };

      // Sign the token
      const token = jwt.sign(payload, JWT_SECRET, {
        algorithm: "HS256"
      });

      return {
        success: true,
        token: token,
        member: {
          id: member._id,
          email: member.loginEmail,
          name: payload.name
        }
      };
    } catch (error) {
      console.error("Error generating token:", error);
      return { success: false, error: error.message };
    }
  }
);

/**
 * Verify a token (optional - for backend verification)
 */
export const verifyMemberToken = webMethod(
  Permissions.Anyone,
  (token) => {
    try {
      const decoded = jwt.verify(token, JWT_SECRET, {
        algorithms: ["HS256"],
        issuer: "marylandforwardparty.com",
        audience: "elections.marylandforwardparty.com"
      });
      return { valid: true, member: decoded };
    } catch (error) {
      return { valid: false, error: error.message };
    }
  }
);
```

## Part 2: Frontend "Go to Elections" Button

Add this to your Wix page where you want the "Go to Elections" button:

**Option A: Using Wix Velo on a page**

```javascript
// Page Code (e.g., Home page)
import { generateMemberToken } from "backend/jwt";

$w.onReady(function () {
  // Configure the button
  $w("#goToElectionsBtn").onClick(async () => {
    try {
      // Show loading state
      $w("#goToElectionsBtn").disable();
      $w("#goToElectionsBtn").label = "Loading...";

      // Generate token from backend
      const result = await generateMemberToken();

      if (result.success) {
        // Redirect to elections app with token
        const electionsUrl = `https://elections.marylandforwardparty.com?token=${encodeURIComponent(result.token)}`;
        wixLocation.to(electionsUrl);
      } else {
        // Handle error - maybe user isn't logged in
        if (result.error === "Not logged in") {
          // Redirect to login page with return URL
          const currentPage = encodeURIComponent(wixLocation.url);
          wixLocation.to(`/account/login?returnUrl=${currentPage}`);
        } else {
          $w("#errorText").text = "Error: " + result.error;
          $w("#errorText").show();
        }
      }
    } catch (error) {
      console.error(error);
      $w("#errorText").text = "Something went wrong. Please try again.";
      $w("#errorText").show();
    } finally {
      $w("#goToElectionsBtn").enable();
      $w("#goToElectionsBtn").label = "Go to Elections";
    }
  });
});
```

**Option B: Using a simple link with token generation**

If you want a direct link approach, create a lightweight page that generates the token and redirects:

```javascript
// Page Code for a dedicated "redirect" page
import { generateMemberToken } from "backend/jwt";
import wixLocation from "wix-location";

$w.onReady(async () => {
  const result = await generateMemberToken();
  
  if (result.success) {
    const electionsUrl = `https://elections.marylandforwardparty.com?token=${encodeURIComponent(result.token)}`;
    wixLocation.to(electionsUrl);
  } else {
    // Show login prompt
    $w("#message").text = "Please log in to access the elections system.";
    $w("#loginBtn").show();
  }
});
```

## Part 3: Wix Package Installation

Install the `jsonwebtoken` package in your Wix site:

1. Go to **Code Packages** in your Wix Editor
2. Click **Install Package**
3. Search for `jsonwebtoken`
4. Install version 8.x or 9.x

Alternative: Use Wix's built-in crypto if you prefer not to add dependencies:

```javascript
// Using Wix crypto instead of jsonwebtoken
import { crypto } from "wix-crypto";

function createSimpleToken(payload, secret) {
  const header = JSON.stringify({ alg: "HS256", typ: "JWT" });
  const encodedHeader = base64UrlEncode(header);
  const encodedPayload = base64UrlEncode(JSON.stringify(payload));
  
  const signature = crypto.hmac(
    "sha256",
    secret,
    `${encodedHeader}.${encodedPayload}`
  );
  
  return `${encodedHeader}.${encodedPayload}.${signature}`;
}
```

## Part 4: Security Considerations

### 1. Use Wix Secrets Manager (Recommended)

Don't hardcode the JWT secret. Use Wix Secrets:

```javascript
import { Secrets } from "wix-secrets-backend";

async function getJwtSecret() {
  try {
    const secret = await Secrets.getSecret("jwt_signing_key");
    return secret;
  } catch (error) {
    console.error("Failed to get secret:", error);
    throw error;
  }
}
```

To set up:
1. Go to **Settings > Secrets Manager** in your Wix dashboard
2. Create a new secret named `jwt_signing_key`
3. Use a strong random string (at least 32 characters)

### 2. Short Token Expiration

Tokens expire in 1 hour by default. Adjust based on your needs:
- Shorter = more secure, but user may need to re-login
- Longer = more convenient, but higher risk if stolen

### 3. HTTPS Only

Ensure both sites use HTTPS. Wix provides this by default.

### 4. Token Storage

The HTML app stores the token in localStorage. This is acceptable for:
- Short-lived tokens (1 hour)
- Non-sensitive operations
- Same-domain scenarios (subdomain)

For higher security, consider:
- httpOnly cookies (requires backend coordination)
- Session storage only (no persistence)
- Refresh token pattern

## Testing the Flow

1. **On Wix main site:**
   - Log in as a member
   - Click "Go to Elections" button
   - Verify you're redirected with a token in the URL

2. **On elections app:**
   - The token should be captured and stored
   - Your name should appear in the header
   - The URL should be cleaned (token removed)

3. **Logout test:**
   - Click "Log Out" on elections app
   - Should clear localStorage and show "Log In" button

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Not logged in" error | Ensure user is logged into Wix before clicking button |
| Token not recognized | Check that JWT secrets match between Wix and HTML app |
| CORS errors | Ensure proper headers if calling APIs between sites |
| Token expired | Reduce token expiry or implement refresh mechanism |
| Name not showing | Check that member profile has first/last name in Wix |

## Alternative: Wix Members API Direct Integration

If you prefer not to use JWTs, you could:

1. Store member info in Wix Database
2. Generate a simple one-time token (UUID)
3. Store mapping in database: `token -> memberId`
4. Elections app exchanges token for member info via API

This is more complex but gives you full control over the auth flow.
